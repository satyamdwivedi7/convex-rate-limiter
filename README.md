# convex-rate-limiter

SaaS billing hard-cap limiter for [Convex](https://convex.dev) apps.

Enforce weighted credit budgets per tenant so API spend cannot exceed plan limits.

## Install

```bash
npm install convex-rate-limiter
```

## Wire Up

In your app's `convex/convex.config.ts`:

```ts
import { defineApp } from "convex/server";
import rateLimiter from "convex-rate-limiter/convex.config.js";

const app = defineApp();
app.use(rateLimiter);
export default app;
```

## Why this exists

`@convex-dev/rate-limiter` is excellent for generic traffic shaping. This package is intentionally focused on **cost governance**:

- weighted operation costs (not just 1 request = 1 unit)
- strict hard caps by tenant plan
- explicit budget errors for upgrade/paywall flows

## Component Layout

This package follows Convex component root layout:

- `convex.config.ts`
- `schema.ts`
- `rateLimits.ts` (public/internal budget enforcement functions)
- `utils.ts` (validation + window parsing)
- `billing.ts` (pricing catalog + credit calculation)
- `crons.ts` (daily cleanup cron)
- `_generated/` (component server/api/component typings)

## Usage

Component functions are accessed via the `components` namespace generated by Convex in the host app.

```ts
import { action, query } from "./_generated/server";
import { components } from "./_generated/api";
import { v } from "convex/values";
```

### Configure tenant plan

```ts
await ctx.runMutation(components.rateLimiter.rateLimits.upsertTenantPlan, {
  tenantId: "tenant_123",
  planId: "starter",
  creditLimit: 10_000,
  periodStart,
  periodEnd,
});
```

### `authorizeSpend` â€” charge weighted credits and enforce hard cap

Throws `ConvexError({ code: "BUDGET_EXCEEDED", required, remaining, resetAt })` if the spend would exceed budget.

```ts
export const callModel = action({
  args: { tenantId: v.string(), prompt: v.string() },
  handler: async (ctx, args) => {
    await ctx.runMutation(components.rateLimiter.rateLimits.authorizeSpend, {
      tenantId: args.tenantId,
      operation: "llm.gpt4o.input",
      units: 120,
    });
    // proceed with upstream API call
  },
});
```

Map budget exceed to upgrade/paywall:

```ts
export const protectedAction = action({
  args: { tenantId: v.string() },
  handler: async (ctx, args) => {
    try {
      await ctx.runMutation(components.rateLimiter.rateLimits.authorizeSpend, {
        tenantId: args.tenantId,
        operation: "search.query",
        units: 1,
      });
    } catch (e: any) {
      if (e.data?.code === "BUDGET_EXCEEDED") {
        return { ok: false, reason: "upgrade_required", ...e.data };
      }
      throw e;
    }
    return { ok: true };
  },
});
```

### `peekBudget` â€” read current budget status

```ts
export const getBudget = query({
  args: { tenantId: v.string() },
  handler: async (ctx, args) => {
    return await ctx.runQuery(components.rateLimiter.rateLimits.peekBudget, {
      tenantId: args.tenantId,
    });
  },
});
```

## API Reference

All public component functions use object-style syntax, import builders from `./_generated/server`, and include explicit `args` + `returns` validators.

### `authorizeSpend(ctx, { tenantId, operation, units })`

Returns:

| Field | Type | Description |
|---|---|---|
| `allowed` | `boolean` | `true` when charged successfully |
| `charged` | `number` | Credits consumed for this call |
| `remaining` | `number` | Credits left in active period |
| `resetAt` | `number` | Epoch ms when period ends |

Throws `ConvexError` with `code: "BUDGET_EXCEEDED"` if not enough credits remain.

### `peekBudget(ctx, { tenantId })`

| Field | Type | Description |
|---|---|---|
| `limit` | `number` | Total budget for period |
| `used` | `number` | Credits consumed in period |
| `remaining` | `number` | Credits left |
| `resetAt` | `number` | Period end timestamp |

### `upsertTenantPlan(ctx, { tenantId, planId, creditLimit, periodStart, periodEnd })`

Creates or updates a tenant's active budget policy for a period.

## Migration from request-count API

If you used older request-window functions (`checkRateLimit`, `enforceRateLimit`, `peek`), switch to:

- `authorizeSpend` for spend authorization
- `peekBudget` for status
- `upsertTenantPlan` for plan configuration

## How It Works

Each authorized operation consumes weighted credits from the tenant's period budget atomically. If required credits exceed remaining credits, the operation is denied deterministically with `BUDGET_EXCEEDED`.

This keeps external API usage spend inside hard limits for each tenant plan.
